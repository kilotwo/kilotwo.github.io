1.类集框架简介

在实际项目开发中基本都会遇到数据结构的使用，但是如果每次都需要开发者自己去实现链表等那么开发效率太低，所以Java提供了一种专门解决数据结构的开发框架————类集框架。类集框架在整个Java开发中最为核心的作用就是实现了动态对象数组的操作，并且定义了大量的操作标准，核心接口有：Collection,List,Set,Map,Iterator,Enumeration等。

2.单对象保存父接口：Collection

java.util.Collection是进行单对象保存的最大父接口，即每次利用Collection接口都只能保存一个对象信息，单对象保存的定义如下：

从定义中可以看出该接口中使用了泛型，这样可以保证集合中操作数据类型的统一，同时该接口属于Iterable的子接口。
Collection接口的核心方法如下表：

方法名称 描述
public boolean add(E e) 向集合里面保存数据
public void clear() 清空集合，根元素为null
public boolean isEmpty() 判断集合是否为空
public int size() 取得集合中保存元素的个数
public Iterator 为Iterator接口实例化

一般平时使用较多的是add和Iterator两个方法，虽然Collection是单对象集合操作的最大父接口，但是该接口无法区分保存的数据是否重复，所以一般开发中往往使用它的两个子接口：List接口（数据允许重复）和Set（数据不允许重复）接口，而一般List接口使用最多。

##3. 最常用的 List子接口
List接口最大的功能就是保存的数据可以重复，同时也对Collection进行了一些扩充，扩充的方法如下：

public E set(int index ,E element） 修改指定索引编号的内容
public ListIteratorlistIterator 为ListIterator接口实例化

在使用List接口时可以利用ArrayList或者Vector两个子接口来进行接口对象的实例化操作

Array中保存的内容是固定的，而ArrayList中保存的内容是可变的，很多时候ArrayList的操作需要判断但是Arrays只需要操作索引即可。
如果是在已经确定好长度的前提下完全可以使用Arrays，如果集合的长度不确定则使用ArrayList。

5.集合输出

集合的输出一般采用循环的方式完成，在Java中集合的输出方式有4中形式：Iterator输出、ListIterator输出、foreach输出、Enumeration输出。

5.1 迭代输出 Iterator

Iterator迭代器是集合输出操作较为常用的接口，任何接口类型都可以转换为 Iterator输出。
在Iterator中一共定义了两种抽象方法

方法 描述
public boolean hasNext() 判断是否还有内容
public E next() 取出当前内容

当使用Iterator输出时，往往都先利用hasNext()改变指针位置，同时判断是否有数据，如果当前指针所在位置存在数据，则利用next()取出数据，上一篇文章提到过的Scanner类扫描器其实就是迭代器的子类，所以需要先使用hasNext来判断是否有数据，再利用hasNext()取得数据。

6.偶对象保存：Map接口

前面所说的Collection每次只能保存一个对象，属于单值保存父接口，而保存偶对象就要使用Map集合，利用Map结合可以保存一对关联数据（按照key=value的形式）。
Map常用方法如下：

方法 描述

public V get (Object key) 根据key查找对应的value数据

那么Collection和Map的区别是什么呢？
很明显Collection是为了输出，Map是为了查询，前者可以使用Iterator输出，而后者可以用key来查找value的字典功能，虽然Map也可以输出但是一般不这么用。另外一般不建议使用自定义类型作为Map的key，其中String作为Key是最为常见的。

##小结

  1. 类集的目的是创建动态对象数组的操作
  2. Collection接口是类集中最大的单值操作父接口，但是一般开发中不使用此接口，而常常使用List和Set接口
  3. List接口扩展了Collection接口，里面的内容是允许重复的，常用的子类是ArrayList,性能较高，是异步处理
  4. 集合的输出要依靠迭代器Iterator接口完成，Iterator属于迭代输出接口
  5. Map可以存放成对的内容，所有的内容都有自己的关键字以key->value的形式保存，常用的子类是HashMap，属于处理，性能较高。
